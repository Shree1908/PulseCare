PulseCare
Project Overview

The system allows patients to book appointments, doctors to manage consultations, and administrators to manage hospital operations. 
The project demonstrates a strong understanding of Spring Boot architecture, authentication mechanisms, database design, and role-based access control. 
This project also implements JWT authentication, OAuth2 login, RBAC authorization, and advanced database operations using JPQL and native queries.

Technologies Used
Java Spring Boot, Spring Security, Spring Data JPA, Hibernate, Authentication & Security, JWT (JSON Web Token), OAuth2 Login, RBAC (Role Based Access Control), Maven.

Database PostgreSQL, JPQL Queries, Native SQL Queries.

Authentication & Authorization JWT based authentication, Secure login system, OAuth2 login integration, Role Based Access Control (RBAC), Protected API endpoints using Spring Security.

Environment / Requirements Java 21+, Spring Boot: 3.x, PostgreSQL: 16, Maven: 3.x,

How to Run the Project 1 Clone the Repository

2 Navigate to the Project cd hospital-management-system

3 Configure Database Update application.properties spring.datasource.url=jdbc:postgresql://localhost:5432/HospitalManagement spring.datasource.username=your_username spring.datasource.password=your_password


This project demonstrates a strong foundational understanding of Spring Boot and modern backend development practices.
It follows the MVC (Model–View–Controller) architecture to maintain a clean and scalable application structure. 
The system exposes multiple RESTful APIs for managing hospital operations such as patients, doctors, and appointments. 
It also implements secure authentication and authorization mechanisms, including JWT-based login, OAuth2 authentication, and Role-Based Access Control (RBAC) to protect application endpoints.
